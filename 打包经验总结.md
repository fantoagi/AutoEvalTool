## 环境准备

### ✅ 必须使用虚拟环境

**原因：**
- 避免系统 Python 环境污染
- 确保依赖版本一致
- 避免 Streamlit 导入卡住问题

**步骤：**

```powershell
# 1. 创建虚拟环境
python -m venv venv

# 2. 激活虚拟环境
.\venv\Scripts\Activate.ps1
# 或
venv\Scripts\activate.bat

# 3. 安装依赖
pip install streamlit pandas openpyxl pyinstaller
```

### ⚠️ 常见错误

**错误1：Streamlit 导入卡住**
- **现象：** `import streamlit` 时程序卡住，Ctrl+C 无法中断
- **原因：** 
  - Streamlit 配置文件损坏
  - Python 进程残留
  - 网络连接问题
- **解决：**
  1. 运行 `强制清理.bat` 结束所有 Python 进程
  2. 删除 `%USERPROFILE%\.streamlit` 目录
  3. 重启计算机
  4. 使用虚拟环境重新安装

**错误2：虚拟环境激活失败**
- **现象：** `'venv\Scripts\activate.bat' is not recognized`
- **原因：** 虚拟环境创建不完整
- **解决：**
  1. 删除旧的 `venv` 目录
  2. 重新创建：`python -m venv venv`
  3. 验证：检查 `venv\Scripts\activate.bat` 是否存在

## 常见问题及解决方案

### 🔴 问题1：RuntimeError: server.port does not work when global.developmentMode is true

**错误信息：**
```
RuntimeError: server.port does not work when global.developmentMode is true.
```

**原因：**
- PyInstaller 打包环境中，Streamlit 默认启用开发模式
- 开发模式下不允许使用 `--server.port` 参数

**解决方案：**

在 `run_gui.py` 中：

1. **创建配置文件，禁用开发模式：**

```python
config_content = """[global]
developmentMode = false

[server]
port = 8501
address = "127.0.0.1"
headless = true
enableCORS = true
enableXsrfProtection = false
runOnSave = false
enableStaticServing = true

[browser]
gatherUsageStats = false
serverAddress = "127.0.0.1"
serverPort = 8501
"""
```

2. **设置环境变量：**

```python
os.environ['STREAMLIT_GLOBAL_DEVELOPMENT_MODE'] = 'false'
os.environ['STREAMLIT_SERVER_PORT'] = '8501'
os.environ['STREAMLIT_SERVER_ADDRESS'] = '127.0.0.1'
```

3. **在用户目录和临时目录都创建配置文件：**

```python
# 用户目录（Streamlit 默认查找位置）
user_home = os.path.expanduser('~')
user_config_dir = os.path.join(user_home, '.streamlit')
user_config_file = os.path.join(user_config_dir, 'config.toml')

# 临时目录（PyInstaller 解压目录）
temp_config_dir = os.path.join(base_dir, '.streamlit')
temp_config_file = os.path.join(temp_config_dir, 'config.toml')

# 在两个位置都创建
for config_file in [user_config_file, temp_config_file]:
    with open(config_file, 'w', encoding='utf-8') as f:
        f.write(config_content)
```

### 🔴 问题2：404 NOT FOUND 错误

**错误信息：**
```
404 GET / (127.0.0.1) 0.45ms
404 GET /favicon.ico (127.0.0.1) 0.39ms
```

**原因：**
- Streamlit 服务器启动失败（通常是问题1导致的）
- 端口被占用
- 配置文件未正确读取

**解决方案：**
1. 先解决问题1（禁用开发模式）
2. 检查端口占用：`netstat -ano | findstr ":8501"`
3. 确保配置文件在正确位置

### 🔴 问题3：app.py 文件找不到

**错误信息：**
```
FileNotFoundError: [Errno 2] No such file or directory: 'app.py'
```

**原因：**
- PyInstaller 未将 `app.py` 包含到打包文件中
- `.spec` 文件配置不正确

**解决方案：**

在 `AutoEvalTool.spec` 中：

```python
# 确保 app.py 被包含
datas = [
    (app_py_path, '.'),  # 将 app.py 复制到临时目录根目录
    (config_dir, 'config'),
    (prompt_dir, '辅助小结'),
] + streamlit_datas
```

### 🔴 问题4：Streamlit 元数据缺失

**错误信息：**
```
importlib.metadata.PackageNotFoundError: No package metadata was found for streamlit
```

**原因：**
- PyInstaller 未正确收集 Streamlit 的元数据文件

**解决方案：**

在 `AutoEvalTool.spec` 中：

```python
from PyInstaller.utils.hooks import collect_all

# 收集 streamlit 的所有数据（包括元数据）
tmp_ret = collect_all('streamlit')
streamlit_datas = tmp_ret[0]
streamlit_binaries = tmp_ret[1]
streamlit_hiddenimports = tmp_ret[2]

hiddenimports = [
    'streamlit',
    'importlib.metadata',
    'importlib_metadata',
    'pkg_resources',
] + streamlit_hiddenimports
```

### 🔴 问题5：浏览器兼容性错误（Object.hasOwn is not a function）

**错误信息：**
```
TypeError: Object.hasOwn is not a function
```

**原因：**
- Streamlit 1.28+ 使用了 ES2022 的 `Object.hasOwn()` 方法
- 目标电脑的浏览器版本过旧（Chrome < 93, Edge < 93, Firefox < 92）
- 无法升级浏览器的情况

**解决方案：**

#### 方案1：降级 Streamlit 版本（推荐）

**步骤：**

1. **运行兼容脚本：**
   ```batch
   兼容旧浏览器.bat
   ```

2. **或手动降级：**
   ```powershell
   pip uninstall streamlit -y
   pip install "streamlit>=1.26.0,<1.27.0"
   ```

3. **修改 requirements.txt：**
   ```
   streamlit>=1.26.0,<1.28.0
   ```

4. **重新打包：**
   ```batch
   .\打包应用.bat
   ```

#### 方案2：添加 Polyfill（已集成）

应用已自动注入 `Object.hasOwn` polyfill（在 `app.py` 中），但可能不够早。

**如果 polyfill 不够早，可以：**
1. 使用浏览器扩展注入 polyfill
2. 或使用用户脚本（如 Tampermonkey）

#### 方案3：使用便携版浏览器

如果无法升级系统浏览器，可以使用便携版：
- Chrome Portable
- Firefox Portable
- Edge Portable

**版本兼容性：**

| Streamlit 版本 | ES2022 支持 | 兼容旧浏览器 |
|----------------|------------|------------|
| 1.28.0+ | ✅ 使用 | ❌ 需要 Chrome 93+ |
| 1.27.x | ⚠️ 可能使用 | ⚠️ 部分兼容 |
| 1.26.x | ❌ 不使用 | ✅ 兼容 Chrome 92- |

**推荐：** 使用 Streamlit 1.26.x 版本

**错误信息：**
```
TypeError: Object.hasOwn is not a function
```

**原因：**
- Streamlit 使用了 ES2022 的 `Object.hasOwn()` 方法
- 目标电脑的浏览器版本过旧，不支持此方法
- 需要 Chrome/Edge 93+, Firefox 92+, Safari 15.4+

**解决方案：**

1. **升级浏览器（推荐）**
   - Chrome/Edge: 菜单 → 帮助 → 关于，自动更新
   - Firefox: 菜单 → 帮助 → 关于 Firefox，自动更新

2. **使用其他浏览器**
   - 下载最新版 Chrome 或 Edge
   - 使用便携版浏览器（如 Chrome Portable）

3. **在文档中说明浏览器要求**
   - 在 README.md 中添加浏览器要求
   - 创建 `浏览器兼容性说明.md` 详细说明
   - 在启动时显示提示信息

**预防措施：**
- 在分发 exe 时附上 `浏览器兼容性说明.md`
- 在 `使用说明.txt` 中说明浏览器要求
- 在 `run_gui.py` 启动时显示浏览器要求提示

### 🔴 问题6：端口冲突（3000 vs 8501）

**现象：**
- 程序显示 `http://localhost:3000`，但无法访问
- 实际服务在 8501 端口

**原因：**
- Streamlit 的 Node dev server 尝试使用 3000 端口
- 但 Node.js 在打包环境中不可用

**解决方案：**

1. **禁用 Node dev server：**

```python
os.environ['STREAMLIT_SERVER_ENABLE_STATIC_SERVING'] = 'true'
```

2. **在配置文件中：**

```toml
[server]
enableStaticServing = true
```

3. **使用 127.0.0.1 而不是 localhost：**

```python
os.environ['STREAMLIT_SERVER_ADDRESS'] = '127.0.0.1'  # 避免 IPv6 问题
```
